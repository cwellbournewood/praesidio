# ISO/IEC 27001:2022 — Annex A control mapping

Selected Annex A controls most relevant to an AI inference control plane.

| Control | Title | Section |
|---|---|---|
| A.5.7 | Threat intelligence | Detector rule updates via signed bundles; SIEM feed |
| A.5.15 | Access control | OIDC, RBAC/ABAC |
| A.5.23 | Information security for use of cloud services | Per-upstream model registry + jurisdiction routing |
| A.5.33 | Protection of records | Hash-chained audit + signed exports |
| A.5.34 | Privacy & PII protection | Anonymiser + retention policy |
| A.8.2 | Privileged access rights | RBAC role `admin` is separate; just-in-time elevation supported via OIDC group claims |
| A.8.3 | Information access restriction | RLS in Postgres, Redis key prefix |
| A.8.10 | Information deletion | Subject delete RPC; vault TTL |
| A.8.11 | Data masking | Anonymiser per-finding transforms |
| A.8.12 | Data leakage prevention | the whole platform |
| A.8.15 | Logging | Audit + lineage |
| A.8.16 | Monitoring activities | Prometheus + dashboards; per-tenant FP rate panels |
| A.8.23 | Web filtering | Network egress allow-list on agent sandbox |
| A.8.24 | Cryptography | AES-256-GCM, Ed25519, TLS 1.3, FF3-1; key rotation runbook |
| A.8.28 | Secure coding | CI: ruff, mypy, bandit, semgrep; dependency review |
| A.8.32 | Change management | Signed bundles + simulation + canary |
